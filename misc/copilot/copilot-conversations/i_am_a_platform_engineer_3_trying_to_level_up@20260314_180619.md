## Why IoC? 
Solves **tight coupling**: Makes code **modular**, **testable**, **flexible**, **extensible**.

## IoC vs [[Dependency Injection]] (DI)
| Concept | What It Is | Example |
|---------|------------|---------|
| **IoC** | Principle: "Let framework control" | Spring calls your `@Controller` |
| **DI**  | Pattern: Inject deps (one IoC form) | `@Autowired UserService` |

## Common IoC Forms
- **[[Dependency Injection]]** (Most popular)
  ```java
  // ✅ GOOD: Framework injects
  public class UserController {
      private final UserService service;

          this.service = service;
      }
  }

  // ❌ BAD: Tight coupling
  public class UserController {
      private final UserService service = new UserService();
  }
